将私有代码托管到git@osc

代码有做版本管理的需求。否则哪天手残删了项目，或者电脑硬盘出了问题，哭都没处哭。

非开源项目，github托管要收费，git@osc的免费托管私有项目是不错的选择。

我现在使用的方法是：

  1. 在git@osc上建立私有项目
  2. 将刚建立的私有项目git clone
  3. cd进入clone后出现的文件夹
  4. 将已有项目目录内的所有文件粘贴到该文件夹（如pom.xml等）
  5. 在文件夹内建立并认真编写.gitignore
  6. 将该文件夹导入ide（之前先将老项目从ide中删除），检查无误后继续
  7. git add .
  8. git commit -m "init"
  9. git push

怎样将一个已有的项目文件夹，用git@osc做版本管理？不知道，摸索过几次，有误操作将项目删除的惨痛经历，再不尝试。

现在的做法是在osc网页上建立新库，git clone出现新目录，将项目拷贝进去，再提交。ide导入这个新的目录作为工作项目。

至于原来的项目目录，作为一个以防万一的备份留着也不错，反正之后再也不用它了，ide中也不保留这个路径，之后的修改、提交都与它无关。

现在用的.gitignore：

/target
/.settings
/.classpath
/.project
/.gitignore
